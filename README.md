## Project Overview

**Paper Summarizer** is a pure JavaScript frontend tool for fetching academic paper metadata from Zotero and generating multilingual summaries and analyses using Mistral (mistral-small-latest).

**Core Features:**
- Zotero data synchronization: supports user/group libraries, retrieving title, abstract, authors, year, DOI, etc.
- Paper summary generation: generates three-part summaries (general overview, methodology, dataset description) for each paper
- Tag management and filtering: displays tags by frequency level with multi-select filtering support
- Keyword network graph: force-directed algorithm visualizing keywords and their co-occurrence relationships
- Local caching: papers and summaries stored in browser LocalStorage

**Technology Stack:**
- Frontend: HTML + Bootstrap 5 + vanilla JavaScript (framework-free)
- Data sources: Zotero Group ID, Zotero API, Mistral API (model: mistral-small-latest)
- Deployment: static server only

**Use Cases:**
Research group literature management, classroom teaching, rapid literature review browsing.

## Current architecture

- Frontend: single-page [index.html](index.html)
- UI: HTML + Bootstrap
- Runtime: vanilla JavaScript in browser
- Data: JSON files in [data](data) and [summaries](summaries)
- Optional local config: [config.local.js](config.local.js) (not committed)

No Node.js backend is required.

## Tools & Transparency

- Metadata: Zotero API (Zotero Group/User libraries)
- DOI/metadata enrichment: Crossref API (optional)
- AI model for summaries: Mistral — `mistral-small-latest` (all generated paper summaries saved under the `summaries/` folder were produced by this model)
- Frontend UI: Bootstrap 5, vanilla JavaScript
- Local runtime: browser LocalStorage for caching; static server (http-server or Python http.server) for serving files

AI-generation notes:
- Files that contain AI-generated content (outputs) — `/summaries/*.json` — each summary entry includes a `source` and `model` field (e.g., `"source": "mistral", "model": "mistral-small-latest"`).
- Runtime scripts that call the AI model: `index.html` (client-side summarization flow), `mistral_summarizer.py` and `src/mistralClient.js` (helpers/wrappers). These scripts send prompts to `mistral-small-latest` and save the returned text as summaries.
- No entire source-code files are generated by the AI model; only the paper summaries and textual outputs are produced by Mistral.

If you used any other AI assistance to produce documentation or code comments, please add an explicit note here identifying the file and the model used.

## Aperçu du projet

**Paper Summarizer** est un outil frontend en JavaScript pur pour récupérer les métadonnées des articles académiques depuis Zotero et générer des résumés et analyses multilingues par Mistral (mistral-small-latest).

**Fonctionnalités principales:**
- Synchronisation des données Zotero : supporte les bibliothèques utilisateur/groupe, récupère le titre, résumé, auteurs, année, DOI, etc.
- Génération de résumés : génère des résumés en trois parties (aperçu général, méthodologie, description de l'ensemble de données) pour chaque article
- Gestion des étiquettes et filtrage : affiche les étiquettes par niveau de fréquence avec support de filtrage multi-sélection
- Graphe de réseau de mots-clés : algorithme force-directed visualisant les mots-clés et leurs relations de co-occurrence
- Mise en cache locale : articles et résumés stockés dans LocalStorage du navigateur

**Pile technologique:**
- Frontend : HTML + Bootstrap 5 + JavaScript vanilla (sans framework)
- Sources de données : Zotero Group, ID API Zotero, API Mistral (modèle : mistral-small-latest)
- Déploiement : serveur statique uniquement
